мессенджер с возможностью шифрования сообщений.

клиентская часть: AvaloniaUI, mvvm паттерн, ввиду отстуствия как таковых страниц в авалонии,
итерфейс на пользовательских контролах.

серверная часть: aspnetcore, signalr для организации real-time взаимодействия, сервисы для сообщений,
чатов, отношений между пользователями. доступ к сервисам может быть получен только при успешной валидации пользовательского jwt-токена.
при регистрации или авторизации в системе пользователю выдается токен, с которым он может обращаться к методам signalr хаба.
валидный jwt-токен хранится у пользователя только в рамках одной локальной сессии.
при успешном входе в систему клиент и сервер обмениваются публичными ключами. ключи сереализуются/десереализуются в json-файлы.
предполагаемо конфиденциальная информация шифруется посредством публичных ключей и расшифровывается на стороне получателей.

механизм отправки сообщений: 
1. сообщение на стороне клиента шифруется публичным ключом хаба, чтобы при передаче никто не смог получить доступ к тексту
2. на стороне хаба сообщение расшифровывается приватным ключом хаба
3. новое сообщение шифруется алгоритмом aes
4. вектор инициализации генерируется рандомно и хранится в таблице с сообщением по причине общей публичности
5. ключ шифрования для сообщения формируется исходя из комбинации уникальных внутренних ключей, единожды полученных пользователями при регистрации
6. ключ шифрования в полном виде нигде не хранится. вектор инициализации генерируется как рандомное число
7. на стороне хаба сообщение шифруется публичным ключом получателя
8. на стороне получателя вызывается событие о получении нового сообщения в том случае, если клиент онлайн (в идеале - находится в этот момент в чате)
9. получатель расшифровывает сообщение своим приватным ключом
10. пользовательский интерфейс клиентов синхронно обновляется

библиотека под бд: code-first подход, связи между сущностями через fluentAPI. orm - entity framework core, провайдер - postgreSQL

бибилиотека под операции шифрования: bigint-товый rsa для обмена зашифрованными сообщениями, aes для шифрования сообщений внутри бд.

хэширование - стандарт формирования ключа на основе пароля (pbkdf),  кол-во итераций - 10000, алгоритм - sha512, соль также генерируется случайная.
